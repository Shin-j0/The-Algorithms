# 💻The Algorithms
## 🚀 소개
### 프로젝트 소개
이 프로젝트는 백준 온라인 저지(BOJ) 문제를 체계적으로 학습하고,
해결 과정과 사고 흐름은 문서화하고, 정답 코드는 깔끔하게 분리 관리하기 위해 설계된 개인 알고리즘 아카이빙 프로젝트입니다.

단순히 “맞힌 문제 수”를 늘리는 것이 아니라,

왜 이 알고리즘을 선택했는지

시간·공간 복잡도를 어떻게 줄였는지

다른 접근과의 차이점은 무엇인지

를 명확히 남기는 것을 목표로 합니다.

### 주요 기능
알고리즘 공부를 하다 보면 다음과 같은 문제가 반복됩니다.

풀이 코드는 남아 있는데, 왜 그렇게 풀었는지 기억이 안 남

GitHub에 코드와 설명이 뒤섞여 가독성이 떨어짐

문제 수가 많아질수록 정리·탐색이 어려워짐

포트폴리오로 보여주기엔 맥락이 부족함

이 프로젝트는 이를 해결하기 위해
“코드(Solutions)와 설명(Writeups)을 명확히 분리” 하고,
자동화된 동기화 구조를 통해 지속 가능한 알고리즘 기록 시스템을 제공합니다.

### 기술적 특징
Writeups Repository

문제 해설, 접근 전략, 핵심 아이디어

알고리즘 분류 / 난이도 / 날짜 기반 정리

Notion → GitHub 자동 동기화

Solutions Repository

순수 소스 코드만 관리 (C++ / Python / Java 등)

문제 번호 기준 파일 구조

자동 커밋 메시지 규칙 적용

👉 설명은 읽기 좋게, 코드는 보기 좋게
👉 사람이 보는 문서와 컴파일러가 보는 코드를 분리

### 프로젝트 구조
🧠 사고 과정 중심 기록

“정답”이 아닌 “접근”을 남김

📚 알고리즘 분류 기반 아카이빙

BFS / DFS / DP / Greedy / Graph 등

🤖 자동화된 워크플로우

Notion → GitHub Actions → Markdown

🧩 확장 가능한 구조

언어 추가, 문제 수 증가에도 구조 유지

🧑‍💻 실전 코딩 환경 기반

VS Code + MinGW + Git 실전 세팅

## 사용 방법

### 1. 저장소 구조

- `baekjoon-writeups`
  - 문제 풀이 설명 (Markdown)
  - 알고리즘 분류 / 난이도별 정리
- `baekjoon-solutions`
  - 실제 제출용 소스 코드 (C++, Python 등)

### 2. 풀이 문서 열람

본 저장소는 **정적 문서 저장소**입니다.  
별도의 빌드나 실행 과정 없이 GitHub에서 바로 열람할 수 있습니다.

👉 문제별 풀이 문서는 `docs/baekjoon/` 디렉토리에서 확인할 수 있습니다.
